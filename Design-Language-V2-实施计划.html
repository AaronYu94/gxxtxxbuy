把"堆卡片"重构为
一条 物流 Journey。

买家端已具备工程完成度，但缺少统一的设计哲学。这份计划围绕一个核心概念重建整个视觉系统：一件商品从中国出发，经采购、质检、仓储、合包、空运，最终送达全球——整站都是这条路线上的一个位置。

§ 背景与现状

为什么要做 V2，而不是打磨按钮

整站是"白底 + 一模一样的圆角卡片"堆叠：信息密度平均、没有视觉高潮、没有品牌记忆点、几乎没有 Motion，插画/照片/3D/UI 四种视觉语言混用。工程 ≈ 70–80%，但停留在模板阶段。技术底座已探明，决定了实施方式：

纯静态 · 无构建单个 styles.css(3181 行) + 经典 <script>；render 函数返回模板字符串。加 <link>/内联 <svg> 都 trivial（OAuth 品牌标已有先例）。
Token 极弱仅 17 个颜色变量 + 1 个 shadow，约 192 处硬编码 hex 绕过；完全没有间距/圆角/字阶/Motion token。
组件同质几乎所有面 = 白 + 1px line + radius 6–8 + shadow（border-radius 出现 61 次）——这就是"半成品感"的根源。
Motion ≈ 01 个 @keyframes、1 个 transition、1 个真实 hover-lift；无 IntersectionObserver/rAF；无 prefers-reduced-motion。
三份步骤清单并存journey(8 项)、内联 process(6 项)、guide 时间线(8 项)必须收敛为一份。
孤儿资产可退役mascot.png / wordmark.jpg / logistics-hero.jpg 零引用；hero 背景照片将随"照片退役"移除。
交付形态：先做自包含原型 app/home-v2.html（内联 tokens + SVG + 一段 IIFE motion），截图反复迭代定稿后，再抽成 styles-v2.css 叠加、把 markup 接进 renderDashboard。插画统一为极简几何线+面 SVG，彻底放弃照片+3D 混搭。

§ 四条不动摇的设计法令

贯穿所有工作包

01Journey 就是设计系统任何面都是路线上的一个位置/检查点，不是通用"卡片"。
02照片退役唯一插画语言 = 几何 SVG 路线套件；无照片、无 3D 羊、营销区无 UI 截图。
03蓝/紫各有职责蓝 = 境内/仓储/追踪，紫 = 国际空运，珊瑚 = 品牌 & "你在这里"。没有装饰性色彩。
04六卡 → 一条脊柱把六张几乎相同的 process 卡换成一条连续 Journey 脊柱——最高杠杆的单点改动，也是页面视觉高潮。
05叠加覆盖，不重写新增 styles-v2.css 挂在老文件之后覆盖 token，第一轮绝不改 3181 行 styles.css。

§C · Journey SVG 品牌套件

内联 · 几何 · 可动（stroke-width 1.8 对齐 lucide）

统一坐标语言：基于 stroke、round cap、24×24 字形网格、纯几何，填色来自 token。会动的一律手写内联 SVG；纯静态的可用 lucide。

  • Route Spine — Hero 主角：缓 S 形路径 + 7 节点；底轨 warehouse + 上层 coral 用 stroke-dashoffset 画入；节点带 halo 脉冲。
  • Package — 3 个平行四边形拼的准等距箱 + 珊瑚胶带；单 <g id="parcel"> 供 JS 沿路径 getPointAtLength() 平移。
  • Warehouse — 梯形屋顶 + 卷帘线，ship-blue。Plane — 纸飞机三角 + 画入的虚线尾迹，air-purple。
  • 7 个阶段节点字形（link/cart-check/scan/warehouse/layers/plane/home-pin）：静态可用 lucide，Hero 关键三件手写。
§D · 卡片系统 5 级

终结"20 张一样的卡"

Primaryr16 · 无边 · e-2 · 珊瑚顶条 · lift 4px
Secondaryr10 · 1px line · e-1 · lift 2px
Ghostr10 · 虚线 · 无阴影 · 空态/添加
Floatingr24 · 无边 · e-3 · 滚动视差
Inliner6 · 无阴影 · tint · chip/tile

硬约束：每屏最多一个 Primary + 一个 Floating，其余只能 Secondary/Ghost/Inline。这一条就足以打破同质。

§E · Motion 系统

无框架：CSS + IntersectionObserver + rAF；必须尊重 reduced-motion

Journey 载入画入 · package 沿路滑行 · 节点脉冲 · 滚动揭示(共享 IO 切 .is-in) · 时间线自动推进(setInterval，hover 暂停) · 按钮 lift · 卡片 hover 2–4px。prefers-reduced-motion 在脚本顶部短路：加 .rm 类 → 关所有动画并强制 .is-in，package 停在 Delivered。
