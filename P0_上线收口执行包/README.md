# GOATEDBUY P0 上线收口执行包

> 目标：在 V1.0 正式上线前，先完成会影响用户数据安全、核心链路真实性、页面可信度和上线稳定性的 P0 收口事项。

## P0 定义

P0 不是完整功能上线，而是上线前必须清掉的红线风险：

- 用户不能看到别人的订单、地址、QC、包裹、wallet 数据。
- 前端不能展示 mock 商品、测试订单、测试账号或假状态。
- 订单、QC、包裹、wallet 状态必须和后端真实字段一致。
- 首页不能出现官方 finds、商品推荐货架或平台商品背书。
- 服务费、国际运费、coupon 限制、退款/不可退规则不能被隐藏。
- creator 只能看到归因/转化所需数据，不能看到用户敏感数据。
- 日志不能记录完整地址、token、支付信息。

## 推荐执行顺序

1. 产品先过 `02_字段与状态映射表.md`，确认所有用户可见状态和文案。
2. 后端和安全过 `03_权限与数据安全检查表.md`，确认接口、日志、导出和后台权限。
3. 前端、后端、QA 过 `04_mock与测试数据清理表.md`，确认生产无假数据。
4. PM 拉齐 `05_不稳定模块降级与隐藏清单.md`，决定哪些模块本期隐藏或降级。
5. QA 按 `01_P0_主控台.md` 回归核心链路。
6. 所有人在 `06_P0_上线签核表.md` 完成 Go / No-Go。

## P0 核心链路

```text
Home
-> Paste Link
-> Link Intake
-> My Haul
-> Orders
-> QC Center
-> Shipping / Parcel
-> Wallet / Coupon
-> Trust Center
```

## P0 验收口径

| 验收项 | Go 标准 | No-Go 情况 |
| --- | --- | --- |
| 权限隔离 | 用户只能访问自己的数据。 | 任意越权访问、接口可枚举他人数据。 |
| 敏感字段 | 接口只返回页面所需字段，日志脱敏。 | 返回完整地址、token、支付信息或 creator 不该看的用户信息。 |
| 状态真实 | 页面状态和后端字段一致。 | 前端写死状态、状态码不匹配、异常状态缺失。 |
| mock 清理 | 生产无假商品、假订单、测试账号、假 QC。 | 生产页面可见测试/假数据。 |
| 首页定位 | 只做外部链接承接和流程解释。 | 出现官方 finds、商品推荐货架、平台背书。 |
| 费用透明 | 服务费、运费 estimate/final、coupon 限制可解释。 | 隐藏费用、承诺不确定费用、coupon 规则不清。 |
| 降级可控 | 不稳定模块隐藏或有明确兜底。 | 半成品模块暴露给用户。 |
