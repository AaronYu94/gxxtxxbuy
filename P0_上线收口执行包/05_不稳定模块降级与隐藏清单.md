# 05 不稳定模块降级与隐藏清单

> 用途：上线时间紧时，确保未打通、字段不完整或存在风险的模块不会半成品暴露给用户。

## 降级原则

- 核心链路必须可跑通；非核心但有风险的功能可以隐藏。
- 字段不稳定时，优先展示保守状态和最后更新时间。
- 操作流程未打通时，不展示可点击按钮，只展示说明或联系客服。
- 费用、退款、coupon、物流承诺不确定时，不能展示精确承诺。
- 隐藏功能不等于删除需求，需要记录进入 V1.1/V1.2 的条件。

## 首页定位检查

| 检查项 | 结论 |
| --- | --- |
| 是否展示官方 finds | 必须为否 |
| 是否展示官方商品推荐货架 | 必须为否 |
| 是否对第三方 creator 商品做平台背书 | 必须为否 |
| 是否说明链接来自站外，平台负责履约 | 必须为是 |
| 是否有 Trust Center / disclaimer 入口 | 必须为是 |

## 费用与承诺降级

| 场景 | 允许展示 | 禁止展示 |
| --- | --- | --- |
| 运费预估 | estimated shipping、可能变化原因、体积重说明 | 未称重前承诺 final price |
| 物流时效 | estimated delivery window、线路说明 | 保证到达日期 |
| coupon | 可用范围、有效期、不可用原因 | 无条件可用、支付失败不回滚 |
| 退款 | 退款状态和规则说明 | 未确认的精确到账承诺 |
| QC | 图片和用户可选下一步 | 对真伪、品牌、购买价值做背书 |

## P0 No-Go 条件

- 状态字段未对齐但仍上线 Orders / QC / Shipping 真实操作。
- 运费接口不稳定但页面展示确定 final cost。
- QC 决策流未打通但展示可点击 Approve / Return 按钮。
- creator landing page 可访问用户敏感数据。
- Trust Center 没有基础规则或 disclaimer。
- 首页出现官方 finds 或商品推荐货架。
