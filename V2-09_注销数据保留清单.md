# V2-09-13 用户注销数据保留与匿名化清单

注销执行入口：`account-repository.processNextDeletion()`（V2-02-09 落地，V2-09-13 验证）。
一次事务内完成，写 `audit_logs(action='account.anonymize')`。

## A. 匿名化清除的个人数据（PII）

| 表 | 字段 | 处理 |
| --- | --- | --- |
| `users` | `email` / `email_normalized` | 置为 `deleted+{user_id}@anonymous.invalid`（每用户唯一） |
| `users` | `display_name` | 置空 |
| `users` | `password_hash` | 置 `disabled`（不可登录） |
| `users` | `phone` / `phone_verified_at` | 置 null |
| `users` | `country_code` | 置 null |
| `users` | `status` | 置 `banned`（鉴权层拒绝任何请求） |
| `users` | `anonymized_at` / `deleted_at` | 置 now() |
| `addresses` | 收件人/电话/国家/地区/城市/邮编/line1/line2 | 覆盖为 `Deleted user` / 空 / `ZZ` / `Deleted` / `-` / `Deleted`；`deleted_at` 置位 |
| `email_verification_tokens` | 整行 | 删除 |
| `user_devices` | 整行 | 删除 |
| `sessions` | `revoked_at` | 置位（现有会话立即失效） |

## B. 依法/审计保留的数据（不得删除）

| 表 | 保留原因 |
| --- | --- |
| `order_parents` / `item_orders` | 交易与履约审计、财税凭证 |
| `ledger_transactions` / `ledger_entries` | 双分录资金账本不可变（充值/支付/退款/提现/佣金全链） |
| `after_sales_orders` / `after_sales_history` / `after_sales_bills` | 售后与退款审计 |
| `consolidation_parcels` / `parcel_bills` | 包裹与国际运费账单审计 |
| `audit_logs` | 全操作留痕（含本次 `account.anonymize`） |
| `account_status_history` | 账户状态变更历史 |

> 资金与订单表以 `user_id` 关联，注销只匿名化 `users`/`addresses`，不级联删除这些审计表，
> 故资金账本重算与订单履约审计在注销后仍完整。

## C. 验证不变量（V2-09-13 回归）

1. 注销后 `users.status='banned'`、`password_hash='disabled'`、`anonymized_at` 非空 → **无法登录**。
2. `order_parents` / `ledger_entries` 记录数在注销前后不变 → **资金/订单审计完整**。
3. 原邮箱释放（users 邮箱已改为 `anonymous.invalid`）→ **同邮箱可按策略重新注册**。
4. `addresses` 收件人被覆盖为 `Deleted user` → **地址 PII 已清除**。
