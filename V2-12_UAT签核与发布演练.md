## V2-12-26 生产发布与回滚演练

### A. 发布清单(各组件独立可回滚)

| 组件 | 发布方式 | 回滚方式 | 前置 |
| --- | --- | --- | --- |
| 前端静态(三端) | CDN 版本切换 | 切回上一版本目录 | 构建产物哈希留存 |
| API(backend) | 蓝绿/金丝雀镜像 | 切回旧镜像 tag | `/health` `/ready` 通过 |
| DB migration | `npm run migrate`(前置、向后兼容) | 向后兼容迁移或 PITR 恢复 | migration-lint 无未审批破坏性、checksum 无漂移 |
| 队列 worker | 滚动重启 | 暂停消费+切回旧版 | 死信为 0/可控 |
| 支付配置 | secret manager 切换 | 切回旧密钥 | 回调 smoke 通过 |
| 物流配置 | 配置版本切换 | 切回旧配置版本 | 轨迹 smoke 通过 |

### B. 灰度与回滚

1. 迁移前置:DB 迁移必须向后兼容(旧代码可读新 schema),先迁移后发 API。
2. 金丝雀:API 先放 5% 流量,观察 5xx 错误率(<1%)、支付回调、队列死信。
3. 独立回滚:前端/API/DB/队列/支付/物流各自可单独回滚,互不阻塞。
4. DB 回滚:优先向后兼容迁移;不可回滚的破坏性变更走 PITR(见基础设施规格 RPO/RTO)。

### C. 值班与真实 smoke

- 发布后立即运行 smoke(`tests/release-smoke.test.js` 覆盖 `/health` `/openapi.json` `/openapi/events`、受保护路由拒匿名、前台读):
  - [ ] `/health` = 200 ok
  - [ ] `/openapi.json` 全资源接口(>100 paths)
  - [ ] 受保护后台路由拒匿名(401/403)
  - [ ] 关键业务 smoke:下单/充值/签收各跑一次真实用例
- 值班:发布窗口双人值守 30 分钟,盯错误率/回调/队列告警(见基础设施规格监控表)。
