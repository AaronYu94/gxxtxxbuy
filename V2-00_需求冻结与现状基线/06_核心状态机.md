# V2-00-06 核心状态机

## 通用规则

- 状态只能通过命名业务动作推进，禁止通用 `PATCH status` 直接写值。
- 每次转换校验当前状态、操作者角色、数据范围、业务前置条件和资源版本。
- 重复提交同一幂等键返回第一次结果，不重复执行副作用。
- 每次转换追加历史：对象、原状态、新状态、动作、原因、操作者、角色、UTC 秒级时间、请求 ID、幂等键和证据引用。
- 人工强制处理仅限超级管理员；采购组长只允许表中标记的受控修正。
- 高风险强制处理必须填写原因、上传凭证、完成 2FA 二次验证，并写不可删除审计。
- 终态不可离开。确需恢复时创建新业务对象或走单独的高风险恢复流程，不覆盖历史。

## 代购商品子订单

字段分为 `fulfillment_status` 和 `exception_status`，避免涨价或缺货覆盖真实履约位置。

### fulfillment_status

| Code | 中文 | 允许的正常下一状态 | 触发动作 |
| --- | --- | --- | --- |
| `pending_payment` | 待支付 | `agent_ordering`、`cancelled` | 钱包支付成功、用户取消或支付超时 |
| `agent_ordering` | 代下单 | `purchasing`、`cancelled` | 自动分配采购账号、无法分配取消 |
| `purchasing` | 采购处理中 | `seller_dispatch_pending`、`cancelled` | 采购确认购买、异常取消退款 |
| `seller_dispatch_pending` | 已下单待发货 | `seller_dispatched`、`cancelled` | 采购登记商家发货、商家取消 |
| `seller_dispatched` | 商家已发货 | `arrived` | 仓库扫描国内单号 |
| `arrived` | 已到仓 | `qc_in_progress` | 仓库领取并开始 QC |
| `qc_in_progress` | QC 中 | `warehoused` | 固定照片、测量和检查完成 |
| `warehoused` | 已入库 | `parcel_reserved`、`return_in_progress`、`destroy_pending` | 用户提交包裹、退货审核通过、仓储超期 |
| `parcel_reserved` | 已加入包裹 | `warehoused`、`outbound` | 打包前取消释放、包裹出库 |
| `return_in_progress` | 退货处理中 | `refunded`、`warehoused` | 平台退款完成、退货撤销或拒绝后恢复 |
| `destroy_pending` | 待销毁 | `destroyed` | 仓库扫码并上传销毁证据 |
| `outbound` | 已出库 | `completed` | 国际包裹签收并完成 |
| `completed` | 已完成 | 无 | 终态 |
| `cancelled` | 已取消 | 无 | 终态；如已付款必须有关联退款 |
| `refunded` | 已退款 | 无 | 终态 |
| `destroyed` | 已销毁 | 无 | 终态 |

### exception_status

| Code | 中文 | 允许动作与结果 |
| --- | --- | --- |
| `none` | 无异常 | 可创建涨价、缺货、规格不可购或采购失败异常。 |
| `price_change_pending` | 待处理涨价 | 用户补款后回 `none`；取消后进入退款；24 小时未响应自动取消。 |
| `availability_pending` | 缺货/规格不可购待确认 | 用户可等待、更换规格、更换链接或取消；处理后回 `none` 或进入退款。 |
| `customer_material_pending` | 待用户补资料 | 用户补充后回原处理节点；超时按异常规则关闭。 |
| `refund_pending` | 待退款 | 钱包退款成功后为 `resolved`，同时主状态进入 `cancelled` 或 `refunded`。 |
| `manual_review` | 人工异常处理 | 仅责任岗位和上级可处理；不得自动跳过资金或仓库前置条件。 |
| `resolved` | 异常已解决 | 只读历史；新异常创建新记录，不复用旧记录。 |

## 转运预报与转运商品

| Code | 中文 | 允许的下一状态 |
| --- | --- | --- |
| `declared` | 待预报/已预报 | `seller_dispatched`、`cancelled` |
| `seller_dispatched` | 商家已发货 | `arrived` |
| `arrived` | 已到仓 | `qc_in_progress` |
| `unclaimed` | 待认领 | `arrived`、`manual_exception` |
| `qc_in_progress` | QC 中 | `warehoused` |
| `warehoused` | 已入库 | `parcel_reserved`、`destroy_pending` |
| `parcel_reserved` | 已加入包裹 | `warehoused`、`outbound` |
| `outbound` | 已出库 | `completed` |
| `completed` | 已完成 | 无 |
| `cancelled` | 已取消 | 无 |
| `destroy_pending` | 待销毁 | `destroyed` |
| `destroyed` | 已销毁 | 无 |
| `manual_exception` | 人工异常 | 由仓库组长受控恢复到明确业务节点 |

转运商品不允许进入自动退货状态机，只能由客服建立人工处理记录。

## QC 任务

| Code | 中文 | 允许的下一状态 |
| --- | --- | --- |
| `pending` | 待 QC | `claimed` |
| `claimed` | 已领取 | `in_progress`、`pending` |
| `in_progress` | QC 中 | `exception`、`completed` |
| `exception` | QC 异常 | `in_progress`、`completed`、`cancelled` |
| `completed` | 已完成 | 无；重拍创建修订记录，不改原完成时间 |
| `cancelled` | 已取消 | 无 |

`completed` 的前置条件为到货测量完成、四类普通照片齐全、已购额外照片或详细检查全部完成。完成时间同时成为正式入库时间和五日退货起点。

## 在库商品

| Code | 中文 | 允许的下一状态 |
| --- | --- | --- |
| `in_stock` | 在库 | `reserved`、`return_reserved`、`destroy_pending` |
| `reserved` | 包裹占用 | `in_stock`、`picking` |
| `picking` | 拣货中 | `packing`、`in_stock`、`exception` |
| `packing` | 打包中 | `outbound`、`exception` |
| `return_reserved` | 退货占用 | `returning`、`in_stock` |
| `returning` | 退货中 | `returned`、`exception` |
| `destroy_pending` | 待销毁 | `destroyed` |
| `outbound` | 已出库 | 无 |
| `returned` | 已退商家 | 无 |
| `destroyed` | 已销毁 | 无 |
| `exception` | 仓库异常 | 仓库组长修正至记录的目标状态 |

同一库存单元只能存在一个有效占用。仓位不能通过状态接口修改，必须执行商品码和仓位码双扫码动作。

## 国际包裹

| Code | 中文 | 允许的下一状态 |
| --- | --- | --- |
| `draft` | 草稿 | `packing_fee_due`、`cancelled` |
| `packing_fee_due` | 待支付打包及增值服务费 | `warehouse_acceptance_pending`、`cancelled` |
| `warehouse_acceptance_pending` | 待仓库接单 | `picking`、`cancelled` |
| `picking` | 拣货中 | `packing`、`exception` |
| `packing` | 打包中 | `shipping_fee_due`、`exception` |
| `shipping_fee_due` | 待支付国际运费 | `outbound_pending` |
| `outbound_pending` | 待出库 | `outbound`、`exception` |
| `outbound` | 已出库 | `in_transit` |
| `in_transit` | 运输中 | `delivered`、`exception` |

| `completed` | 已完成 | 无 |
| `cancelled` | 已取消 | 无 |

取消规则：仓库执行 `start_packing` 前可取消并释放库存占用、退还可退费用；开始打包后禁止用户取消、增减商品或修改服务。

| Code | 中文 | 允许的下一状态 |
| --- | --- | --- |
| `draft` | 待装批次 | `loading`、`cancelled` |
| `loading` | 扫描装批 | `handoff_pending`、`exception` |
| `handoff_pending` | 待交接 | `handed_off`、`exception` |
| `handed_off` | 已交接 | `completed`、`exception` |
| `completed` | 已完成 | 无 |
| `cancelled` | 已取消 | 无 |

## 资金对象

### 充值单

`created -> pending_provider -> succeeded`

异常分支：`pending_provider -> failed | expired | exception`。`succeeded` 为终态，重复回调不得再次入账。

### 钱包账本交易

`pending -> posted` 或 `pending -> failed`。已 `posted` 记录不可修改或删除；冲正必须创建方向相反且引用原交易的新交易，状态为 `reversed_by_reference` 仅作派生展示。

### 提现单

`pending_review -> processing -> succeeded`

拒绝分支：`pending_review -> rejected`；失败分支：`processing -> failed`。申请时冻结，`rejected` 或 `failed` 时通过独立解冻交易恢复。

### 调账申请

`pending_review -> approved -> executed`

其他分支：`pending_review -> rejected`；`approved -> execution_failed`。财务发起，超级管理员审批，审批人不得等于发起人。

### 费用账单

`draft -> payable -> paid`；取消为 `cancelled`，退款为 `refund_pending -> refunded`。打包费账单和国际运费账单必须是两个独立账单。

## 售后退货

| Code | 中文 | 责任角色 | 允许的下一状态 |
| --- | --- | --- | --- |
| `purchase_review_pending` | 待采购审核 | 采购 | `purchase_reviewing` |
| `purchase_reviewing` | 采购审核中 | 采购 | `customer_material_pending`、`rejected`、`return_fee_due`、`warehouse_picking_pending` |
| `customer_material_pending` | 待用户补充 | 用户 | `purchase_reviewing`、`closed` |
| `return_fee_due` | 待支付退货费用 | 用户 | `warehouse_picking_pending`、`closed` |
| `warehouse_picking_pending` | 待仓库拣货 | 仓库 | `return_verifying` |
| `return_verifying` | 退货复核中 | 仓库 | `return_packing`、`exception` |
| `return_packing` | 退货打包中 | 仓库 | `merchant_return_pending`、`exception` |
| `merchant_return_pending` | 待寄回商家 | 仓库/采购 | `returned_to_merchant` |
| `returned_to_merchant` | 已寄回商家 | 采购 | `merchant_refund_pending`、`exception` |
| `merchant_refund_pending` | 商家签收待退款 | 采购 | `platform_refund_pending`、`exception` |
| `platform_refund_pending` | 商家已退款待平台退款 | 财务 | `completed`、`exception` |
| `completed` | 已完成 | 系统 | 无 |
| `rejected` | 已拒绝 | 采购 | 无 |
| `closed` | 已关闭 | 系统 | 无 |
| `exception` | 异常 | 当前责任岗位 | 返回明确合法节点或关闭 |

客服只能查询和沟通，不能批准、拒绝或修改售后状态。

## 验收规则

- 每个状态均有唯一 code、中文标签、责任角色和合法出口。
- 自动化测试应遍历允许转换和拒绝所有未列转换。
- 状态历史不可被详情表的当前状态替代。
- 并发动作使用资源版本或数据库锁，只有一个动作成功。
