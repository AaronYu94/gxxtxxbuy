# V2-00-07 角色权限与数据范围矩阵

## 固定角色

| Code | 中文角色 | 固定数据范围 |
| --- | --- | --- |
| `super_admin` | 超级管理员 | 全系统全量数据 |
| `procurement_agent` | 采购人员 | 仅分配给本人采购账号的任务 |
| `procurement_lead` | 采购组长 | 本采购组织范围 |
| `support_agent` | 客服 | 只能通过用户 ID、邮箱、订单号或包裹号进入明确业务对象 |
| `warehouse_operator` | 仓库人员 | 分配给本人的任务及完成动作所需关联数据 |
| `warehouse_lead` | 仓库组长 | 本仓库全量数据 |
| `finance_operator` | 财务 | 资金、支付、退款、提现、对账和风控关联数据 |
| `campaign_operator` | 活动运营 | 优惠券、轮播图、邮件和必要受众数据 |
| `referral_operator` | 推广运营 | 邀请和佣金运营数据；被邀用户敏感金额与业务明细不可见 |

每个员工账号只能绑定一个角色。新增能力应创建经批准的新组织角色，不允许同一账号叠加多个角色。

## 字段权限

| 字段组 | 可见角色 | 限制 |
| --- | --- | --- |
| 用户邮箱、手机号、地址 | 超管；客服 SEARCH；仓库 RELATED；财务 SEARCH | 仅业务需要字段；推广运营不可见原文。 |
| 密码 hash、session token、TOTP secret | 无后台页面可见 | 只允许安全服务使用；日志永不输出。 |
| 支付卡、银行账户、身份材料 | 财务专用安全页面和超管审批 | 分字段加密；普通用户详情不加载原文。 |
| 钱包余额、流水、退款金额 | 超管、财务 | 客服仅见当前业务付款状态，不见完整流水。 |
| 采购实际成本 | 超管、采购本人、采购组长 | 客服、仓库、运营、推广不可见。 |
| 公司利润和完整营业额 | 超管 | 财务也不可见利润报表。 |
| 被邀用户邮箱 | 推广运营和推广用户只见打码值 | 不见商品、地址、QC、具体支付金额。 |
| 内部备注 | 创建角色、被授权责任角色、超管 | 用户不可见；用户可见备注为独立字段并二次提示。 |
| QC/仓库图片 | 用户本人、仓库、相关采购、搜索进入的客服、超管 | 私有签名 URL；普通角色不可删除。 |

## 搜索与导出

- 客服、活动运营和财务的用户查询必须至少提供用户 ID、完整/前缀邮箱、订单号或包裹号之一。
- 客服和运营空条件请求返回 `400 SEARCH_CRITERIA_REQUIRED`，不能返回默认第一页。
- 前缀邮箱搜索最少 3 个字符，最多返回 20 个明确候选；每次敏感查询写审计。
- 只有超级管理员拥有导出权限。导出需要目的、筛选范围、行数、文件有效期和 2FA，并写审计。

## 服务端执行要求

- 前端隐藏按钮不构成授权。所有 API、文件和异步任务消费者必须重复校验权限及数据范围。
- 资源不存在和无权访问对普通用户统一返回 `404`，避免枚举他人资源；后台明确权限不足可返回 `403`。
- 列表、详情、聚合统计分别应用数据范围，禁止先查全量再在前端过滤。
- 直接 URL、修改 query、替换资源 ID、批量接口和导出均纳入九角色回归测试。
