# V2-00-04 MVP 与完整 V2.0 边界

## 可运营 MVP

MVP 目标是在小流量下完成真实代购与转运闭环。以下能力必须随 MVP 上线：

| 领域 | MVP 必含能力 | 对应任务 |
| --- | --- | --- |
| 身份与权限 | 邮箱注册验证、用户会话、后台单角色、数据范围、2FA、审计 | V2-01 |
| 用户网站 | 账户、语言框架、展示币种、多地址、关键页面三端状态 | V2-02 |
| 商品 | 淘宝、1688、Weidian 合法解析或明确人工降级、不可变快照 | V2-03 |
| 订单采购 | 母子订单、钱包支付、自动采购分配、涨价/缺货、24 小时取消、商家发货 | V2-04 |
| 钱包财务 | CNY 双重账本、充值回调幂等、扣款、退款、普通提现、调账审批、基础对账 | V2-05 |
| 仓库 | 到货扫码、测量、固定四图 QC、库存单元、仓位双扫码、90/150 天规则、销毁 | V2-06 |
| 包裹物流 | 合包占用、10 CNY 打包费、增值服务、拣货复核、最终运费、出库批次、轨迹 | V2-07 |
| 售后 | 代购五日退货、多角色状态机、商家退款、钱包退款 | V2-08 |
| 用户风控 | 用户精确搜索、账户锁定审批、地址风险、注销匿名化 | V2-09 的基础能力 |
| 运营 | 国际运费优惠券、协议公告、事务邮件、按角色首页 | V2-10 的基础能力 |
| 上线保障 | 私有对象存储、队列死信、四环境、备份恢复、监控、安全、14 条 E2E、灰度回滚 | V2-12 |

## MVP 可延后能力

这些能力可以在核心闭环稳定后进入完整 V2.0，但数据库和事件模型不得阻断后续接入：

- 会员高级等级和免费退换货权益。
- 推广佣金高级运营、银行卡提现和复杂违规冲正。
- 营销邮件深度统计、复杂动态分组和营销看板。
- 在线聊天；MVP 先使用邮件工作台。
- 活动统计深化和非关键轮播运营能力。
- 非核心内容社区、Haul Story、Creator Campaign 和 Country Shipping Hub。

## 不得从 MVP 删除的红线

- 钱包账本、支付回调幂等、退款幂等和余额并发控制。
- 后台角色、服务端数据范围、2FA、高风险二次验证和不可删除审计。
- 到货、仓位、拣货、打包的扫码防错和库存占用锁。
- 商品、金额、汇率、路线、地址和配置历史快照。
- 生产备份恢复、私有图片访问、监控告警和发布回滚。
- PRD 定义的核心 E2E、资金一致性、权限和安全回归。

## 完整 V2.0

完整 V2.0 等于 214 项原子任务全部完成，并且包括：

- 会员完整权益。
- 优惠券、轮播图、多语言推广邮件和统计。
- 邀请关系、五级佣金、佣金钱包、转余额、提现和冲正。
- 邮件加在线聊天的客服工作台及响应指标。
- 所有公共配置版本、运营后台、财务对账和稳定性能力。

## 发布门槛

| 发布层级 | 必须满足 |
| --- | --- |
| 内部测试 | P0 决策已批准；核心状态、权限、数据和 API 规范无冲突。 |
| 预发布 | 核心业务全部使用真实或沙箱供应商；无占位商品或伪支付结果。 |
| 小流量 MVP | 严重缺陷为 0；资金、权限、扫码、备份和核心 E2E 通过。 |
| 完整 V2.0 | 214 项完成；业务负责人和技术负责人共同签署 UAT。 |

旧系统数据不迁移。上线时必须明确切换日、旧系统只读入口和客服跨系统查询 SOP。
