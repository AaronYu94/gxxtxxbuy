# V2-00-08 核心数据字典

## 全局字段规范

| 类型 | 规范 |
| --- | --- |
| 内部主键 | UUID；不向普通前端展示。 |
| 业务编号 | 使用 `05_业务编号规范.md`；服务端生成、不可变、唯一索引。 |
| 时间 | 数据库保存 UTC；API 使用 ISO 8601 秒级时间；页面按配置业务时区展示完整日期和秒。 |
| 日期 | 仅纯日历概念使用 `YYYY-MM-DD`，不得混用 UTC 时间。 |
| 金额 | 使用整数最小货币单位；字段后缀 `_minor`；禁止浮点数。 |
| CNY 账本 | CNY 分为最小单位；所有钱包记账使用 `amount_cny_minor`。 |
| 外币快照 | 同时保存 `original_currency`、`original_amount_minor`、`amount_cny_minor`、`amount_usd_minor`、`exchange_rate_id` 和舍入结果。 |
| 汇率 | 十进制定点数，不用二进制浮点；交易完成后不可修改快照。 |
| 数量 | 正整数；单位明确为件、张、克、毫米或厘米。 |
| 重量 | 数据库存克 `weight_grams`，必须大于 0。 |
| 尺寸 | 数据库存毫米；API 可接受厘米但服务端规范化。 |
| 国家/币种/语言 | ISO 3166-1 alpha-2、ISO 4217、BCP 47。 |
| 状态 | 使用 `06_核心状态机.md` 的稳定 code，不保存翻译文案。 |
| 版本 | 可变业务对象带正整数 `version`；写入使用乐观锁或数据库行锁。 |
| 软删除 | 只用于允许删除的配置或个人资料；资金、状态、审计、图片证据禁止删除。 |
| 备注 | `visibility` 区分 `internal` 与 `user_visible`；保存创建人、角色和时间。 |

所有表至少包含 `id`、`created_at`、`updated_at`；需要追责的表增加 `created_by`、`updated_by` 和 `version`。

## 数据分类

| 等级 | 示例 | 处理要求 |
| --- | --- | --- |
| Public | 已发布协议、公开物流路线名称 | 可缓存；不得混入内部字段。 |
| Internal | 配置草稿、采购账号名称、内部备注 | 后台权限控制，不进入普通用户响应。 |
| Confidential | 邮箱、手机号、地址、钱包余额、采购成本 | 字段权限、日志脱敏、传输加密。 |
| Restricted | 银行账户、支付标识、身份材料、TOTP secret | 独立安全域、字段加密、最小权限、访问审计。 |

## Identity

### user

| 字段 | 类型/规则 |
| --- | --- |
| `user_no` | 公开用户编号，随机且唯一。 |
| `email_normalized` | 小写规范化唯一；Confidential。 |
| `email_verified_at` | 可空 UTC 时间。 |
| `phone` | 可空，默认未验证。 |
| `country_code` | ISO 国家代码。 |
| `default_locale` | BCP 47。 |
| `default_currency` | ISO 4217，仅影响展示。 |
| `status` | `normal`、`risk_locked`、`banned`。 |
| `security_locked_until` | 自动安全锁到期时间；不得覆盖人工风控锁。 |
| `anonymized_at` | 注销匿名化完成时间。 |

### admin_user

`employee_no`、姓名、单一 `role_code`、部门/组织、手机号、状态、TOTP 绑定状态、最近登录时间/IP hash。员工停用后会话立即失效，历史日志继续保留员工编号和姓名快照。

### address

关联用户；收件人、电话、国家、州/省、市、邮编、地址行、默认标记、地址规范化 hash、版本。包裹必须复制地址快照，不直接依赖可变地址。

## Catalog Snapshot

### product_snapshot

| 字段组 | 字段 |
| --- | --- |
| 来源 | `platform`、`source_url`、`source_item_id`、`parsed_at`、`source_adapter` |
| 商品 | 标题、主图、店铺、类目 |
| 价格 | 商品原币金额、国内运费原币金额、CNY 快照、汇率快照 |
| 选择 | 规格、尺寸、颜色、数量、SKU 标识 |
| 原始证据 | 安全摘要或 provider response hash，不保存不必要的第三方敏感数据 |
| 完整性 | `complete`、`needs_details`、`failed` 及明确失败码 |

快照创建后不可编辑。人工补录先完成来源标记，再创建新的快照版本；子订单永远引用创建时版本。

## Order/Purchase

### parent_order

`order_no`、用户、提交时间、支付时间、支付状态、子订单数量、商品和国内运费汇总、优惠、实付、多币种快照、第三方交易号。不得保存统一履约完成状态。

### item_order

`item_order_no`、母订单、用户、商品快照、数量、`fulfillment_status`、`exception_status`、采购账号/人员、实际采购金额、采购平台单号、国内物流商/单号、支付/QC/入库/出库关键时间。

### purchase_exception

类型、子订单、原价、现价、差额、原因、截图、可选方案、用户截止时间、用户选择、处理人、处理时间和状态。每个异常独立记录，不覆盖历史。

## Warehouse

### inbound_package

国内物流商/单号、归属用户和订单、到货时间、扫描员工、外包装照片、匹配状态、首次扫描信息。一个国内包裹只允许一个用户和一个订单。

### qc_task

任务编号、商品子订单/转运预报、类型、四图模板、额外照片数量、详细检查项、拆封要求、执行人、开始/完成时间、结果和异常。

### stock_item

`stock_no`、用户、来源对象、QC 完成/入库时间、真实重量和尺寸、运输限制、仓位、90/150 天截止时间、状态、包裹或售后占用引用。

### warehouse_location

区域、货架、层、位置编码、仓库、状态。移动通过独立 `location_movement` 保存原位、新位、原因、扫码员工和时间。

## Parcel/Logistics

### parcel

`parcel_no`、用户、状态、商品数量、申报快照、地址快照、增值服务快照、路线价格版本、最终重量尺寸、计费重量、打包费账单、国际运费账单、优惠券/会员快照、物流商、单号和关键时间。

### parcel_item

包裹、在库商品、占用状态、拣货扫码时间/员工、打包复核时间/员工。对有效占用设置唯一约束。

### shipping_route_version

路线、物流商、国家、限制类型、首重/续重、计费单位、进位、体积重公式、实重取值、操作费、燃油、偏远、包材、保险、时效、生效时间和版本。

### outbound_batch

`batch_no`、物流商、路线、计划/实际交接时间、包裹数、总重量、创建人、交接人员、签字单/接收凭证和状态。

## Wallet/Payment

### wallet

用户、`available_cny_minor`、`frozen_cny_minor`、版本。余额字段是账本投影，不是独立事实来源。

### ledger_transaction / ledger_entry

交易编号、类型、业务引用、幂等键、状态、币种快照、借贷分录、发起人、审批人和冲正引用。所有分录借贷合计必须为零。

### top_up

`top_up_no`、用户、provider、渠道、原币实付、手续费、CNY 到账、系统状态、渠道状态、provider transaction ID、回调校验结果和风险标签。

### withdrawal

`withdrawal_no`、类型、用户、CNY 金额、资金来源、冻结交易、原路退款引用或加密银行卡引用、审核人、付款凭证、状态和失败原因。

### adjustment_request

`adjustment_no`、用户、加/扣款、金额、原因、凭证、关联业务、发起财务、审批超管、2FA 验证引用、执行交易和状态。

## After-sales

### after_sales_case

`after_sales_no`、商品子订单、用户、原因、责任方、运费方、申请数量、用户凭证、五日截止、当前状态和责任角色。

关联明细包括审核记录、退货费用账单、仓库复核/照片、退货地址快照、退货物流、商家退款金额/扣款/凭证、平台退款核算和钱包流水。

## CRM/Promotion/Referral

### support_conversation

会话编号、渠道、外部线程 ID、用户、关联订单/包裹、分类、负责人、状态、首次/最近消息时间和响应指标。消息使用外部消息 ID 去重，附件私有。

### coupon

编码、类型、金额/比例、门槛、最高优惠、国家/路线范围、领取/使用时间、领取后有效天数、每人次数、总量、发放方式和状态。只适用于国际运费。

### referral_relation / commission_transaction

唯一邀请人与被邀请人、绑定来源/时间；佣金交易保存包裹、已批准基数类型、基数金额、等级、比例、佣金金额、状态、签收事件、退款冲正和账本引用。

## 图片和文件

每个文件记录：`file_id`、业务对象、用途、私有 storage key、MIME、字节数、hash、宽高、缩略图 key、上传人/角色/时间、保留策略和安全域。

QC、到货、打包、出库、退货、销毁和凭证图片永久保留，普通角色不得删除。列表使用缩略图，原图通过短时签名 URL 访问；身份材料使用独立 Restricted 安全域。
