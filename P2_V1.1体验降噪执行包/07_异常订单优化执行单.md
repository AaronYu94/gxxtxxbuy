# 07 异常订单优化执行单

> 目标：异常订单不只显示一个状态，而是展示原因、影响、下一步动作和客服入口，降低投诉和反复咨询。

## 页面目标

- 让用户知道异常发生了什么。
- 给用户明确下一步。
- 让采购、客服、用户看到一致状态。
- 把复杂处理导入客服或退款时间线。

## 异常类型

| 异常 | 触发场景 | 用户下一步 |
| --- | --- | --- |
| out_of_stock | 卖家缺货 | 退款 / 换规格 / 联系客服 |
| price_changed | 卖家改价 | 接受差价 / 取消 / 联系客服 |
| seller_no_ship | 卖家长时间不发货 | 继续等待 / 取消 / 联系客服 |
| purchase_failed | 平台采购失败 | 退款 / 联系客服 |
| wrong_spec | 规格可能不一致 | 联系客服 |
| warehouse_issue | 仓库入库/QC 发现问题 | 查看 QC / 联系客服 |
| refund_processing | 退款处理中 | 查看 Refund Timeline |

## 异常卡片结构

| 区域 | 内容 |
| --- | --- |
| 异常标题 | 用户语言，例如 `The seller changed the price.` |
| 原因说明 | 简短解释，不暴露内部备注。 |
| 影响说明 | 是否影响采购、QC、发货或退款。 |
| 下一步动作 | Confirm / Cancel / Contact support / View refund。 |
| 规则入口 | Trust Center 对应锚点。 |
| 更新时间 | 最后更新时间。 |

## 字段要求

| 字段 | 用途 |
| --- | --- |
| `order_id` | 订单标识。 |
| `exception_type` | 异常类型。 |
| `exception_reason` | 用户可见原因。 |
| `internal_note` | 后台内部备注，不返回前端。 |
| `available_actions` | 可执行动作。 |
| `support_ticket_id` | 工单关联。 |
| `refund_id` | 退款关联。 |
| `updated_at` | 更新时间。 |

## 动作策略

| 动作 | P2 要求 |
| --- | --- |
| Accept price change | 如支付差价流程未打通，走客服。 |
| Cancel order | 如自动取消未打通，走客服或退款申请。 |
| Change spec | 如自动换规格未打通，走客服。 |
| View refund | 跳 Refund Timeline。 |
| Contact support | 必须可用。 |

## 降级策略

| 场景 | 策略 |
| --- | --- |
| 异常原因缺失 | 展示 `We need to review this order.` + support。 |
| 动作接口未打通 | 不展示动作按钮，展示 support。 |
| 退款状态未知 | 跳 support，不展示假 Timeline。 |
| 后台备注敏感 | 不返回前端。 |

## 埋点

| 事件 | 时机 | 必填字段 |
| --- | --- | --- |
| `order_exception_view` | 异常卡片曝光 | `exception_type`、`order_status` |
| `order_exception_action_click` | 点击下一步 | `exception_type`、`action_type` |
| `order_exception_support_click` | 点击客服 | `exception_type`、`order_id` |
| `order_exception_resolved` | 异常关闭 | `exception_type`、`resolution_type` |

## 验收标准

- 异常订单必须展示原因和下一步。
- 不暴露内部备注。
- 动作未打通时不展示假按钮。
- 退款异常可跳 Refund Timeline 或 support。
- 客服可以根据 exception_type 快速处理。
