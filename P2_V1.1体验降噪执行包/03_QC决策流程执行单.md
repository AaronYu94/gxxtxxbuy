# 03 QC 决策流程执行单

> 目标：把 QC Center 从“图片展示”升级为“轻量决策流程”，让用户知道看完 QC 后能确认、补拍、退换或联系客服。

## 页面目标

- 展示 QC 图片和商品状态。
- 提供明确下一步动作。
- 减少用户看完 QC 后不知道该做什么的工单。
- 保持平台中立，不对商品真伪、品牌或购买价值背书。

## P2 决策动作

| 动作 | 用户含义 | 系统结果 | 优先级 |
| --- | --- | --- | --- |
| Approve QC | 用户确认 QC 可接受 | 商品进入 Ready to ship 或保持可发货状态 | P0 |
| Request extra photo | 用户需要补拍 | 生成补拍请求，状态变为 Extra photo requested | P0 |
| Return / Exchange | 用户希望退换 | 生成售后/客服请求 | P1 |
| Contact support | 用户需要人工帮助 | 生成工单或进入客服入口 | P0 |
| Read QC policy | 用户查看规则 | 跳 Trust Center QC 锚点 | P0 |

## 状态机

```text
QC pending
-> QC photos ready
-> Approved
-> Ready to ship

QC photos ready
-> Extra photo requested
-> Extra photo ready
-> Approved / Contact support

QC photos ready
-> Return / Exchange requested
-> Support handling
-> Refunded / Replaced / Closed
```

## 字段要求

| 字段 | 用途 |
| --- | --- |
| `item_id` | 商品标识。 |
| `order_id` | 订单关联。 |
| `qc_status` | QC pending / ready / approved / extra_photo_requested / issue_reported。 |
| `qc_photo_urls` | QC 图片。 |
| `qc_action_status` | 用户动作状态。 |
| `extra_photo_status` | 补拍请求状态。 |
| `return_status` | 退换状态。 |
| `support_ticket_id` | 工单关联。 |
| `updated_at` | 状态更新时间。 |

## 交互规则

| 场景 | 规则 |
| --- | --- |
| 无 QC 图片 | 不展示 Approve，状态为 QC pending。 |
| 已 Approve | 禁用 Approve，展示已确认时间。 |
| 已申请补拍 | 禁用重复申请，展示处理中。 |
| 补拍未打通 | 按钮不展示或点击进入 support，不能假装提交成功。 |
| 退换未打通 | 展示联系客服，不展示自动退换成功。 |
| 商品已加入 parcel | 退换动作需要提示可能无法处理。 |

## 文案边界

| 允许 | 禁止 |
| --- | --- |
| `Review the QC photos before shipping.` | `We guarantee this item is authentic.` |
| `Request extra photos if you need more details.` | `This item is worth buying.` |
| `Contact support for return or exchange questions.` | `GOATEDBUY recommends this item.` |

## 埋点

| 事件 | 时机 | 必填字段 |
| --- | --- | --- |
| `qc_action` | 点击 QC 决策 | `action_type`、`item_id`、`qc_status` |
| `qc_approve_success` | Approve 成功 | `item_id`、`order_id` |
| `extra_photo_request_submit` | 提交补拍 | `item_id`、`reason` |
| `return_exchange_start` | 发起退换 | `item_id`、`reason` |
| `qc_support_click` | 点击客服 | `item_id`、`qc_status` |

## 验收标准

- QC ready 后至少有一个明确下一步动作。
- Approve 成功后状态和 My Haul / Shipping 同步。
- 补拍请求不会重复提交。
- 未打通动作有降级，不展示假成功。
- 页面不做商品背书。
