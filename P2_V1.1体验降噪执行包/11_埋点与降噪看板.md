# 11 埋点与降噪看板

> 目标：用数据判断 V1.1 是否真的降低新手疑惑、QC 阻塞、shipping 争议和客服压力。

## 埋点原则

- P2 埋点围绕“解释是否被看到”和“看完后是否减少工单”。
- 不上报完整地址、token、支付信息。
- support 事件必须带来源页面和问题类型。
- FAQ 和 Guide 需要能归因到后续动作。

## 降噪看板

| 指标 | 口径 | 目标方向 |
| --- | --- | --- |
| 首单用户客服咨询率 | first-time 用户工单数 / first-time 用户数 | 下降 |
| First Haul Guide 使用率 | guide view / first-time 用户数 | 提升 |
| Guide 后首单提交率 | guide view 后 purchase_order_submit / guide view | 提升 |
| QC 确认率 | qc_approve_success / qc_view | 提升 |
| 补拍请求率 | extra_photo_request_submit / qc_view | 可监控 |
| QC 后客服率 | qc_support_click / qc_view | 下降 |
| Shipping FAQ 使用率 | shipping_faq_view / shipping_view | 提升 |
| 运费支付成功率 | shipping_pay_success / parcel_submit | 提升 |
| coupon 不可用咨询率 | coupon_issue 工单 / coupon_not_applicable_view | 下降 |
| 异常订单投诉率 | order_exception 工单 / exception order | 下降 |

## 归因分析

```text
Guide/FAQ/Policy exposure
-> User action
-> Support ticket created or avoided
-> Conversion / completion
```

| 分析问题 | 需要字段 |
| --- | --- |
| 看 First Haul Guide 后是否更容易提交首单？ | `first_haul_guide_view`、`purchase_order_submit` |
| QC 按钮是否减少客服？ | `qc_action`、`support_ticket_create` |
| Shipping FAQ 是否提升运费支付？ | `shipping_faq_view`、`shipping_pay_success` |
| coupon 解释是否减少咨询？ | `coupon_not_applicable_view`、`coupon_issue` |
| 异常订单动作是否降低投诉？ | `order_exception_action_click`、`support_ticket_create` |

## 验收标准

- P2 新增事件在测试环境可见。
- support_ticket_create 能区分问题类型和入口。
- 不上报敏感信息。
- 看板至少覆盖首单、QC、shipping、coupon、异常订单五类降噪指标。
- 上线后一周能输出第一版降噪复盘。
