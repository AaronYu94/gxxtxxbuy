# GOATEDBUY PRD V2.0 开发原子任务

> 唯一需求基线：`PRD/GoatedBuy_开发级详细PRD_V2.0.pdf`。旧 B0-B8 和 P0-P4 的完成记录只作为代码复用参考，不自动继承为本清单的完成状态。

## 执行规则

每个任务必须能在一次开发 run 中完成，并同时满足：

- 只解决一个明确问题，避免跨领域连带开发。
- 依赖、交付物和验收条件都可直接检查。
- 代码任务包含成功、空输入、重复请求、越权、状态冲突和外部失败路径中适用的测试。
- 涉及金额时使用数据库事务、幂等键和不可删除流水。
- 涉及状态时校验前置状态并记录状态历史。
- 涉及后台时同时校验角色权限和数据范围。
- 涉及敏感操作时记录审计日志；高风险操作增加 2FA 二次确认。
- migration 可重复执行，并提供向后兼容或明确回滚方式。
- 完成后运行该任务的定向测试及 `npm run ci`，然后更新 `prd-v2-task-status.json`。

## 状态勾选规则

1. 新清单初始全部为未完成。
2. 浏览器手动勾选只保存在本地，不代表代码已经验收。
3. Codex 完成任务并验证通过后，将任务 ID 写入 `prd-v2-task-status.json` 的 `completed`。
4. `notes` 必须记录主要交付物、测试命令和结果。
5. 功能只完成 UI、只完成 API 或只通过 happy path 时不得勾选整项。
6. PRD 参数未冻结导致无法验收时，任务保持未完成，并记录在 `blocked`。
