# 09 工作台信息架构与导航

> 目标：重新组织用户工作台，让 Link Inbox、My Haul、Orders、QC、Shipping、Wallet、History 不互相抢入口，而是围绕用户下一步行动聚合。

## IA 原则

- 工作台首页优先展示用户下一步该做什么。
- 链接、商品、订单、QC、包裹、历史是同一条 haul lifecycle，不应割裂。
- 重要状态聚合到 dashboard，复杂详情进入子页面。
- 移动端导航要少而清楚，避免页面层级过深。

## 推荐导航

| 一级导航 | 子内容 |
| --- | --- |
| Dashboard | 下一步任务、haul 概览、提醒、recent activity。 |
| Link Inbox | saved links、needs details、failed parse。 |
| My Haul | active haul、Haul Builder、ready to ship。 |
| Orders | purchase orders、exceptions、refunds。 |
| QC Center | QC ready、extra photo、approved。 |
| Shipping | parcel drafts、shipping preview、tracking。 |
| Wallet | coupons、credit、Savings Stack。 |
| History | Haul History、past parcels、recap。 |
| Support | tickets、policies、contact。 |

## Dashboard 组件

| 组件 | 内容 |
| --- | --- |
| Next action | 需要用户处理的最高优先级事项。 |
| Saved links | 待处理链接数量。 |
| Ready for QC | QC ready 数量。 |
| Ready to ship | 可发货商品数量。 |
| Parcel drafts | 待提交包裹草稿。 |
| Recent history | 最近完成的 haul。 |
| Savings | 真实已节省金额摘要。 |

## 下一步优先级

| 优先级 | 条件 | CTA |
| --- | --- | --- |
| 1 | 异常订单待处理 | Resolve order issue。 |
| 2 | QC ready 未处理 | Review QC photos。 |
| 3 | ready to ship 商品存在 | Build your parcel。 |
| 4 | parcel draft 待支付 | Pay shipping。 |
| 5 | saved links 需补信息 | Complete item details。 |
| 6 | 空工作台 | Paste a link。 |

## 移动端导航

| 位置 | 内容 |
| --- | --- |
| Bottom nav | Dashboard、My Haul、Shipping、History、Account。 |
| Secondary menu | Link Inbox、Orders、QC、Wallet、Support。 |
| Sticky CTA | 根据下一步动态显示。 |

## 验收标准

- 用户进入 Dashboard 能看到下一步。
- Link Inbox 和 My Haul 边界清楚。
- Ready to ship 能顺畅进入 Haul Builder / Shipping Preview。
- History 不干扰 active haul。
- 移动端导航不超过核心入口负担。
