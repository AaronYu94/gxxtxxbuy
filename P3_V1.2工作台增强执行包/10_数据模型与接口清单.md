# 10 数据模型与接口清单

> 目标：列出 P3 新增对象、核心字段、接口依赖和跨模块一致性要求，方便前后端对齐。

## 新增/增强对象

| 对象 | 用途 |
| --- | --- |
| `saved_link` | Link Inbox 保存链接。 |
| `parcel_draft` | Haul Builder 生成包裹草稿。 |
| `shipping_preview` | 发货预览和线路对比。 |
| `saving_record` | Savings Stack 优惠记录。 |
| `haul_history` | 历史 haul 聚合。 |
| `haul_recap` | 可分享 recap。 |
| `creator_campaign_touch` | creator campaign 触点。 |

## 接口清单

| 接口 | 方法 | 用途 | P3 要求 |
| --- | --- | --- | --- |
| `/saved-links` | GET | 获取 Link Inbox | 只返回当前用户数据。 |
| `/saved-links` | POST | 保存链接 | 支持未解析先保存。 |
| `/saved-links/{id}` | PATCH | 编辑、归档、标签 | 校验用户归属。 |
| `/saved-links/{id}/add-to-haul` | POST | 加入 My Haul | 防重复。 |
| `/haul-builder` | GET | 获取可发货聚合 | 状态与 Orders/QC/Shipping 一致。 |
| `/parcel-drafts` | POST | 创建包裹草稿 | 校验 ready_to_ship。 |
| `/parcel-drafts/{id}` | PATCH | 更新草稿 | 防重复商品。 |
| `/shipping-preview` | POST | 获取线路和费用预览 | 标明 estimate。 |
| `/savings` | GET | 获取 Savings Stack | 不统计 pending/reversed。 |
| `/haul-history` | GET | 获取历史 haul | 隐私字段脱敏。 |
| `/haul-history/{id}` | GET | 历史详情 | 当前用户可见。 |
| `/haul-recaps` | POST | 创建 recap | 默认隐私保护。 |
| `/haul-recaps/{id}` | PATCH | 更新/撤销 recap | 用户可撤销。 |
| `/creator-campaign/touch` | POST | 记录触点 | 不含敏感数据。 |

## 跨模块一致性

| 数据 | 必须一致的模块 |
| --- | --- |
| item 状态 | Link Inbox、My Haul、Orders、QC、Haul Builder。 |
| ready_to_ship | My Haul、Haul Builder、Shipping。 |
| shipping fee | Shipping Preview、Parcel、History。 |
| coupon / credit | Wallet、Checkout、Savings、History。 |
| creator_code | Creator Landing、Link Inbox、Orders、Campaign Tracking。 |
| privacy settings | History、Recap、Share page。 |

## 权限要求

- 所有用户对象必须按 `user_id` 做服务端权限校验。
- 分享页只能展示 recap 白名单字段。
- creator tracking 对 creator 输出必须汇总或脱敏。
- 接口不得返回完整地址、支付信息、token、后台内部备注。

## 验收标准

- 接口字段和前端展示状态一致。
- 所有 P3 对象可追溯 owner。
- saved link、parcel draft、recap 不能越权访问。
- sharing 和 creator tracking 不泄露敏感信息。
