# 09 后台角色权限治理

> 目标：建立后台角色权限矩阵，避免采购、仓库、客服、运营、财务、风控、管理员之间权限过宽或边界不清。

## 权限原则

- 最小权限。
- 按职责授权。
- 高风险操作需要审批或二次确认。
- 权限变更必须审计。
- 导出和完整敏感字段访问必须单独授权。

## 角色矩阵

| 角色 | 可访问 | 不可访问 |
| --- | --- | --- |
| 采购 | 采购订单、商品规格、卖家信息、采购异常。 | 支付敏感信息、完整地址批量导出、creator 财务。 |
| 仓库 | 入库、QC、重量、包裹、仓储状态。 | 用户支付信息、creator 数据、财务报表。 |
| 客服 | 订单摘要、工单、退款状态、必要地址摘要。 | 批量导出完整地址、后台权限管理。 |
| 运营 | CMS、coupon 活动、社群内容、creator campaign 汇总。 | 用户支付信息、完整订单敏感明细。 |
| 财务 | 支付、退款、服务费、coupon 成本、结算。 | 仓库内部操作、内容审核操作。 |
| 风控 | 风险 case、异常订单、违规内容、coupon 滥用。 | 无关财务明细，除非授权。 |
| 管理员 | 全局配置和权限管理。 | 所有敏感操作必须审计。 |

## 高风险权限

| 权限 | 要求 |
| --- | --- |
| 完整地址查看 | 单独授权，记录原因。 |
| 批量导出 | 审批 + 审计。 |
| wallet/credit 调整 | 财务权限 + 审计。 |
| 退款审批 | 财务/客服主管权限。 |
| creator code 暂停 | 增长主管/风控权限。 |
| 权限变更 | 管理员 + 审计。 |

## 权限生命周期

```text
申请
-> 审批
-> 授权
-> 使用审计
-> 定期复核
-> 回收
```

## 定期复核

| 频率 | 内容 |
| --- | --- |
| 每周 | 高风险导出和异常访问。 |
| 每月 | 角色权限和离职/转岗账号。 |
| 每季度 | 管理员权限和审计日志抽查。 |

## 验收标准

- 后台角色权限矩阵确认。
- 高风险权限有审批和审计。
- 离职/转岗账号可及时回收。
- creator、客服、运营不能越权查看敏感数据。
