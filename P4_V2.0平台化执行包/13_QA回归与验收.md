# 13 QA 回归与验收

> 目标：确认 P4 平台化能力在 creator、用户偏好、内容、风控、审计、权限和埋点层面可用且安全。

## 回归前置

| 项目 | 要求 |
| --- | --- |
| P0-P3 | 核心链路、安全、降噪、工作台能力稳定。 |
| 权限矩阵 | 后台角色和 creator 权限已确认。 |
| 测试数据 | creator、profile、story、risk case、audit log 数据准备完成。 |
| 环境 | staging 完整回归，production smoke test。 |

## No-Go 条件

- creator dashboard 泄露用户敏感数据。
- shipping profile、QC preferences、Haul Stories 存在越权访问。
- 风控后台高风险操作无权限控制。
- 审计日志未覆盖导出、权限变更、财务/wallet 调整。
- Haul Stories 无审核/举报/撤销机制。
- Country Hub 或页面文案承诺不确定运费/到达日期。
- P4 看板或埋点暴露敏感信息。
