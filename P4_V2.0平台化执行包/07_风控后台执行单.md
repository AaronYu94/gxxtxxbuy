# 07 风控后台执行单

> 目标：为异常订单、敏感商品、违规内容、coupon 滥用、creator 风险和数据安全事件提供后台处理能力。

## 后台目标

- 聚合高风险事件。
- 支持标记、分派、处理、关闭。
- 支持风险规则和人工审核结合。
- 所有敏感操作进入审计日志。

## 风险类型

| 风险 | 示例 |
| --- | --- |
| order_risk | 异常订单、重复退款、高投诉订单。 |
| item_risk | 敏感/限制商品、疑似违规商品。 |
| content_risk | Haul Stories、社群内容、creator 内容违规。 |
| coupon_abuse | 批量注册、异常 coupon 使用、刷单。 |
| creator_risk | 误导承诺、异常退款率、异常转化。 |
| data_security | 越权访问、敏感字段泄露、异常导出。 |
| payment_risk | 支付失败异常、退款争议。 |

## 后台结构

| 区域 | 内容 |
| --- | --- |
| Risk queue | 风险事件列表、类型、等级、状态。 |
| Case detail | 相关订单、用户、creator、证据、处理历史。 |
| Actions | 标记、分派、暂停、隐藏、退款审核、升级。 |
| Rule config | 规则开关、阈值、白名单、黑名单。 |
| Audit trail | 操作记录。 |

## 状态流转

```text
new
-> triaged
-> assigned
-> investigating
-> action_taken
-> resolved
-> reopened
```

## 操作权限

| 操作 | 角色 |
| --- | --- |
| 查看风险列表 | 风控、管理员、相关运营。 |
| 处理订单风险 | 风控、客服主管、采购主管。 |
| 隐藏内容 | 内容审核、运营主管。 |
| 暂停 creator code | 增长主管、风控。 |
| 调整 coupon 风控 | 风控、财务、管理员。 |
| 查看数据安全事件 | 安全、管理员。 |

## 审计要求

所有操作必须记录：

- 操作人。
- 操作时间。
- 操作对象。
- 操作前后状态。
- 操作原因。
- 关联工单或风险 case。

## 埋点/日志

| 事件 | 时机 | 必填字段 |
| --- | --- | --- |
| `risk_case_created` | 创建 case | `risk_type`、`risk_level` |
| `risk_case_assigned` | 分派 | `risk_type`、`owner_role` |
| `risk_action_taken` | 执行动作 | `risk_type`、`action_type` |
| `risk_case_resolved` | 关闭 case | `risk_type`、`resolution_type` |

## 验收标准

- 高风险事件可进入队列。
- 处理动作有权限控制。
- 敏感操作进入审计日志。
- 风险 case 可追溯处理过程。
- 风控后台不泄露无关敏感数据给非授权角色。
