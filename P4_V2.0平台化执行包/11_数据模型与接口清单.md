# 11 数据模型与接口清单

> 目标：列出 P4 平台化新增对象、核心字段、接口和权限要求，确保 creator、profile、content、risk、audit 能协同。

## 新增/增强对象

| 对象 | 用途 |
| --- | --- |
| `creator_dashboard_metric` | creator 汇总指标。 |
| `shipping_profile` | 用户 shipping 偏好。 |
| `qc_preference` | 用户 QC 偏好。 |
| `country_shipping_rule` | 国家物流规则。 |
| `haul_story` | 用户分享内容。 |
| `risk_case` | 风险 case。 |
| `audit_log` | 审计日志。 |
| `role_permission` | 角色权限配置。 |
| `content_review` | 内容审核记录。 |

## 接口清单

| 接口 | 方法 | 用途 | 权限 |
| --- | --- | --- | --- |
| `/creator/dashboard` | GET | creator 看板 | creator 本人 / 运营 |
| `/shipping-profile` | GET/PATCH | 用户 shipping profile | 用户本人 |
| `/qc-preferences` | GET/PATCH | 用户 QC 偏好 | 用户本人 |
| `/country-shipping-hub` | GET | 国家物流规则 | 公开/登录均可 |
| `/admin/country-shipping-rules` | POST/PATCH | 维护国家规则 | 运营/物流 |
| `/haul-stories` | POST | 创建 story | 用户本人 |
| `/haul-stories/{id}` | GET/PATCH | 查看/编辑/撤销 story | 按隐私等级 |
| `/admin/content-review` | GET/PATCH | 内容审核 | 审核/运营 |
| `/admin/risk-cases` | GET/PATCH | 风险 case | 风控 |
| `/admin/audit-logs` | GET | 审计查询 | 安全/管理员 |
| `/admin/roles` | GET/PATCH | 权限管理 | 管理员 |

## 跨模块一致性

| 数据 | 必须一致的模块 |
| --- | --- |
| creator 汇总 | Creator Dashboard、Campaign Tracking、风险后台。 |
| shipping profile | Shipping Preview、Country Hub、History。 |
| QC preferences | QC Center、仓库后台、Support。 |
| country rules | Country Hub、Shipping Preview、Trust Center。 |
| Haul Stories | History、Recap、内容审核。 |
| risk case | 风控后台、审计日志、客服 SOP。 |
| audit log | 所有后台高风险操作。 |

## 权限要求

- creator dashboard 只返回汇总或脱敏数据。
- shipping profile 和 QC preferences 仅用户本人可见。
- Haul Stories 按 privacy_level 控制可见性。
- risk case 仅授权角色可见。
- audit log 不可被普通后台角色修改或删除。

## 验收标准

- P4 新增对象有 owner、权限和审计策略。
- 接口不返回非必要敏感字段。
- creator、内容、风控、审计数据可以互相关联但不越权。
