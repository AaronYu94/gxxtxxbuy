# 08 审计日志执行单

> 目标：后台敏感操作、数据导出、权限变更、财务操作、风控动作全部留痕，可追溯、可检索、可审查。

## 审计目标

- 追踪谁在什么时候做了什么。
- 支持安全、财务、风控、合规复盘。
- 防止后台滥权和数据泄露。
- 满足 P0-P4 高风险操作的长期治理。

## 必须审计的操作

| 类型 | 操作 |
| --- | --- |
| 用户数据 | 查看完整地址、修改地址、导出用户数据。 |
| 订单数据 | 修改订单状态、退款处理、异常关闭。 |
| QC/仓库 | 删除/替换 QC 图片、修改重量、修改入库状态。 |
| Shipping | 修改线路、费用、tracking、发货状态。 |
| Wallet/Coupon | 发券、作废 coupon、调整 credit、回滚优惠。 |
| Creator | 暂停 code、调整归因、查看 creator 汇总。 |
| 风控 | 创建/关闭 case、隐藏内容、封禁账号。 |
| 权限 | 新增后台账号、改角色、提权、禁用账号。 |
| 导出 | 导出用户、订单、地址、财务数据。 |

## 日志字段

| 字段 | 说明 |
| --- | --- |
| `audit_id` | 审计 ID。 |
| `actor_id` | 操作人。 |
| `actor_role` | 操作人角色。 |
| `action_type` | 操作类型。 |
| `resource_type` | 对象类型。 |
| `resource_id` | 对象 ID。 |
| `before_snapshot` | 操作前摘要，敏感字段脱敏。 |
| `after_snapshot` | 操作后摘要，敏感字段脱敏。 |
| `reason` | 操作原因。 |
| `ticket_id` | 关联工单。 |
| `ip_address` | IP，按安全规则存储。 |
| `created_at` | 时间。 |

## 查询和留存

| 项目 | 要求 |
| --- | --- |
| 查询 | 按 actor、resource、action、时间、风险类型过滤。 |
| 留存 | 按公司合规要求设定，默认不短于关键业务需要。 |
| 防篡改 | 普通后台角色不可修改或删除审计日志。 |
| 导出 | 审计日志导出需高权限和二次确认。 |
| 脱敏 | 日志中敏感字段必须脱敏。 |

## 告警规则

| 场景 | 告警 |
| --- | --- |
| 短时间大量导出 | 通知安全/管理员。 |
| 非工作时间高敏操作 | 标记异常。 |
| 角色频繁变更 | 触发权限审查。 |
| 多次查看完整地址 | 触发数据访问审查。 |
| 手动调整 wallet/credit | 触发财务复核。 |

## 验收标准

- 高风险后台操作都有审计记录。
- 审计日志不可被普通角色篡改。
- 敏感字段脱敏。
- 可按关键维度查询。
- 异常操作有告警或审查机制。
