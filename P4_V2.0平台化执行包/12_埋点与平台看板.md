# 12 埋点与平台看板

> 目标：衡量 V2.0 平台化能力对 creator 留存、用户偏好、内容传播、风险治理和审计覆盖的效果。

## P4 事件表

| 编号 | 事件名 | 页面/后台 | 时机 | 必填字段 |
| --- | --- | --- | --- | --- |
| D-001 | `creator_dashboard_view` | Creator Dashboard | 访问 | `creator_id`、`campaign_count` |
| D-002 | `creator_campaign_detail_view` | Creator Dashboard | 查看活动 | `creator_id`、`campaign_id` |
| D-003 | `shipping_profile_update` | User Profile | 更新偏好 | `updated_field`、`country` |
| D-004 | `shipping_profile_apply` | Shipping Preview | 应用偏好 | `country`、`preferred_line_used` |
| D-005 | `qc_preferences_update` | QC Preferences | 更新偏好 | `preference_type`、`category` |
| D-006 | `country_shipping_hub_view` | Country Hub | 访问 | `country`、`entry_page` |
| D-007 | `haul_story_publish` | Haul Stories | 发布 | `story_id`、`review_status` |
| D-008 | `haul_story_report` | Haul Stories | 举报 | `story_id`、`report_reason` |
| D-009 | `risk_case_created` | Risk Console | 创建 case | `risk_type`、`risk_level` |
| D-010 | `risk_case_resolved` | Risk Console | 关闭 case | `risk_type`、`resolution_type` |
| D-011 | `audit_log_created` | Backend | 生成审计日志 | `action_type`、`resource_type` |

| D-013 | `role_permission_changed` | Admin | 权限变更 | `role`、`permission_type` |

## 平台看板

| 看板 | 指标 |
| --- | --- |
| Creator | creator 留存、campaign 转化、shipping payment、复购、风险标记。 |
| User Profile | profile 创建率、偏好应用率、Shipping Preview 转化。 |
| QC Preferences | 偏好设置率、偏好应用率、补拍率、QC 后客服率。 |
| Country Hub | 国家页浏览、FAQ 展开、Shipping Preview 跳转、客服引用。 |
| Content | Haul Stories 发布数、通过率、举报率、撤销率。 |
| Risk | case 数、风险类型、处理时长、复开率。 |
| Audit | 审计覆盖率、高风险操作数、导出尝试、异常告警。 |

## 关键指标

| 指标 | 目标方向 |
| --- | --- |
| creator 留存 | 提升 |
| creator 复购贡献 | 提升 |
| profile 应用后 shipping 支付率 | 提升 |
| Country Hub 后客服率 | 下降 |
| 内容审核 SLA 达成率 | 提升 |
| 风险 case 平均处理时长 | 下降 |
| 高风险操作审计覆盖率 | 接近 100% |
| 异常导出拦截率 | 可监控 |

## 隐私要求

- creator 看板只用汇总或脱敏数据。
- 埋点不含完整地址、支付信息、tracking、token。
- audit log 中敏感字段脱敏。
- 风控和内容审核看板按角色展示。

## 验收标准

- P4 新增事件在测试环境可见。
- Creator、Risk、Audit 三类看板至少有基础版本。
- 高风险操作能生成 audit_log_created。
- 看板不暴露用户敏感数据。
